管理者ガイド

管理対象

  • Dataverseテーブル
  • SharePointライブラリ
  • Power Automateフロー
  • 外字置換辞書
  • 期限ルール
  • 承認者設定
  • セキュリティロール
  • 監査設定

初期設定

1. Dataverseテーブルを作成する。 2. 代替キーとリレーションを設定する。 3. 監査を有効化する。 4. SharePointライブラリを作成する。 5. Power Appsを作成する。 6. Power Automateフローを設定する。 7. 外字辞書を登録する。 8. 期限ルールを登録する。 9. セキュリティロールを割り当てる。

期限ルール管理

regulatory_deadline_rules で管理する。

例:

変更種別役割期限通知
追加分担医師6か月60,30,14,7,1
削除分担医師6か月60,30,14,7,1
氏名変更責任医師12か月90,60,30,14,7

外字辞書管理

  • 標準縮退マップ
  • 社内辞書
  • 過去届出の置換実績
  • 禁止文字/要確認文字

辞書を更新しても過去届出の確定済み表記は自動変更しない。変更が必要な場合は新しい版を作成する。

承認者管理

  • 起票者と承認者は分離する。
  • 承認後の版は編集不可にする。
  • 代理承認ルールを定義する。
  • 電子署名の意味を記録する。

監査確認

確認対象:

  • 監査が無効化されていないこと
  • 提出後データが変更されていないこと
  • 変更が版として残っていること
  • 出力ジョブと承認版が紐づいていること
  • リマインダー送信履歴が残っていること
