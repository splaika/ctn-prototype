# CTN Suite — プロジェクトメモリ

治験届（CTN: Clinical Trial Notification）管理システム（本番想定UIデモ）。
React + TypeScript + Vite。単一ソース `src/ctn/ctn-schema.json`。デモ本体は
`splaika/ctn-prototype` の `demo/` に同期・公開。

## 【根幹】治験届の番号ルール（治験届の手引き 2024年3月版・東京医薬品工業協会）

> これは業務の根幹ロジック。ここを誤ると生成XMLの届出回数が誤値になり、
> 30日調査や提出期限の判断を誤る。実装・レビュー時は必ずこの原則に従う。

### 構造はツリー（フラットな通し番号ではない）

```
治験成分記号（例 TYK-123 = 固有の医薬品/投与経路の組み合わせ）
├─ 届出回数=1（プロトコール1・計画届）── 変更回数 0 → 1 → 2 …
├─ 届出回数=2（プロトコール2・計画届）── 変更回数 0 → 1 …
└─ 届出回数=3（プロトコール3・計画届）…
```

### 3つの原則

1. **1プロトコール = 1 N回届** — 新規の治験実施計画書（プロトコール）は必ず新しい「届出回数」となる。
2. **届出回数（N回）＝開発全体を追跡する通し番号** — 一つの治験成分記号に対し、何本のプロトコールが存在するかを管理。**新規プロトコールの「治験計画届」を出すたびに +1**（例 1→2→3）。
3. **変更回数＝特定プロトコール内の枝番** — ある届出回数（プロトコール）が何回改訂されたかのバージョン番号（例 0→1→2）。

### 種別ごとの採番（最重要）

| 届出種別 | 届出回数 | 変更回数 |
| --- | --- | --- |
| 治験計画届（**新規プロトコール**＝N回届） | **+1（インクリメント）** | — |
| 治験計画変更届 | **対象プロトコールを継承（据え置き）** | **+1** |
| 治験終了届 / 治験中止届 | 対象プロトコールを継承（据え置き） | — |
| 開発中止届 | 届出回数不要（`"00"`・開発プログラム全体） | — |

- **変更届・終了届・中止届は「届出回数」を増やさない。** 対象プロトコール（届出回数）に紐づき、変更届のみ「変更回数」を採番する。
- 例（TYK-123）: 計画届`届1` → 変更届`届1/変1` → 新第II相 計画届`届2` → 変更届`届2/変1`。「届1」の変更は「届2」に影響しない。
- **N回届 ≠ 変更届。** N回届＝同一成分の新規プロトコール（第I相→新第II相、別適応症等）の計画届。変更届＝既存プロトコールの修正。

### 30日調査（薬機法第80条の2第3項）

- **本邦で初めて人に投与する薬物等**（初回治験計画届・届出回数=1）が対象。届出日から30日経過後でなければ治験を依頼できない。
- **N回届（届出回数≥2の新規プロトコール）は原則30日調査対象外**（例：観察研究）。初回の30日調査終了までN回届の治験は開始不可（同時提出時は「念書」対応）。

### 変更届の提出時期（区分1/2/3ではなく "提出時期" の4分類・手引き p.86-89）

| 提出時期 | 例 |
| --- | --- |
| **変更前** | 治験の目的・対象疾患の変更／30日調査対象の被験薬の追加 |
| **変更後6ヶ月以内** | 実態変更を伴わない製造所名変更／CROの変更・追加・削除／治験届出者の代表者変更 |
| **変更後1年以内** | 治験分担医師の氏名の変更・追加・削除のみ |
| **終了/中止時でOK** | 進捗に伴う予定被験者数の変更／予定交付数量の変更 |

出典: 治験届の手引き（2024年3月版）。当局校閲は経ていない参考資料。
