# CTN作成支援システム 設計概要

## 目的

治験届の初回届、変更届、中止届、終了届、開発中止届を複数製品・複数版で管理し、入力、レビュー、承認、PDF確認、XML/PDF出力、監査証跡、期限リマインドを一貫して扱う。

想定規模:

- 初回届: 20製品以上
- 変更届: 各製品10回以上
- 常時複数ドラフト、レビュー、承認待ちが並行
- 医療機関、医師、治験使用薬、添付資料、外字置換の反復管理

## 基本方針

- Dataverseを正本DBとする。
- SharePointはPDF原本、添付資料、生成済みPDF/XML、外字説明資料の保管に使う。
- Power Appsは入力UIとレビューUIに使う。
- Power Automateは承認、通知、リマインダー、ジョブ起動に使う。
- Azure Functionsまたは同等のサーバー処理でPDF/OCR抽出、XML生成、XSD検証、PDF生成を行う。
- XML/PDF出力は承認後のみ可能にする。
- 入力画面にXMLは常時表示しない。必要時のみプレビューまたは出力結果として確認する。

## 機能ブロック

| ブロック | 主な役割 | 実装候補 |
|---|---|---|
| 入力UI | 届出概要、治験計画、医療機関、医師、治験使用薬などの入力 | Power Apps model-driven app + 必要箇所PCF |
| ダッシュボード | 製品横断、変更届進捗、期限、承認、出力状況 | Power Apps view / Power BI embedded |
| PDF取込 | 既存PDF原本保存、抽出、候補確認、DB化 | SharePoint + Azure Functions + Dataverse |
| ワークフロー | 起票、レビュー、承認、電子署名、出力許可 | Power Automate + Dataverse |
| リマインダー | 6か月/12か月期限、担当者/承認者通知 | Power Automate scheduled flow |
| 外字対応 | 原表記、届出用表記、置換記録、説明資料 | Dataverse + PCF補助 + サーバー検証 |
| 出力 | PDFイメージ確認、XML/PDF生成、XSD検証 | Azure Functions + SharePoint保管 |
| 監査 | 変更履歴、承認履歴、版履歴、出力履歴 | Dataverse audit + 独自監査テーブル |

## 重要な設計判断

- 治験使用薬の順序番号は「突合キー型」とし、計画届から終了届まで詰め直さない。
- 医師の順序番号は「イベント行型」とし、届出ごとに追加、削除、氏名変更を行として表す。
- 医師や組織の同一性は順序番号ではなく、不変IDで管理する。
- 外字は原表記と届出用表記の二段構えで保持する。
- 既存PDFは直接正本化せず、抽出候補を人が確認してから確定版DBへ反映する。
- 変更届は確定済み版から新規ドラフト版を作成し、差分入力する。
