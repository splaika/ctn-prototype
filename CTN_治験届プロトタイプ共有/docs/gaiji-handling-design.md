# 外字対応設計書

## 基本方針

外字は自動置換のみで確定しない。原表記と届出用表記を分け、人が確認した置換記録を残す。

## 二段構え

| 表記 | 説明 |
|---|---|
| 原表記 | 入力またはPDF取込で得た元の表記。原本性のため保持する |
| 届出用表記 | PMDA届出用にJIS範囲などへ正規化した表記 |

## 対象

- 医師名
- 医療機関名
- 代表者名
- 届出者名
- CRO/SMO名
- 海外依頼者名
- 住所
- 添付資料名

## gaiji_mappings

| 列 | 説明 |
|---|---|
| `source_table` | 対象テーブル |
| `source_record_id` | 対象レコード |
| `source_column` | 対象列 |
| `original_text` | 原表記 |
| `submission_text` | 届出用表記 |
| `detected_character` | 検出文字 |
| `code_point` | Unicodeコードポイント |
| `detection_type` | 旧字、機種依存、IVS、PUA、範囲外など |
| `replacement_reason` | 置換理由 |
| `confirmed_by` | 確認者 |
| `confirmed_at` | 確認日時 |
| `attachment_document_id` | 外字説明資料 |

## チェックロジック

1. 入力またはPDF取込時にUnicode文字を検査する。
2. 社内外字辞書と標準縮退マップで候補を提示する。
3. 届出用表記を自動候補として入れる。
4. ユーザーが確認または修正する。
5. 確認済みにならない限り、承認または出力に進めない。
6. 承認後、外字置換説明資料を生成する。

## 実装候補

- Power Apps標準フォームで原表記/届出用表記を表示
- PCFで外字ハイライト、候補提示、コードポイント表示
- サーバー側プラグインで最終検証
- Azure Functionsで外字説明資料生成

## 禁止事項

- 原表記を上書きして消すこと
- 未確認の置換でXML/PDFを正式出力すること
- PCFだけに検証を依存すること
